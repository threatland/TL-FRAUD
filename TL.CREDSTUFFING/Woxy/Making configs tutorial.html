    Configuration Format

    Format Style of Config Making:

    Config making requires basic knowledge of JSON & REGEX.

    {
  "General": {
    "name": "",
    "author": "",
    "description": "",
    "onceEnabled": "false"
  },
  "Capture": [
    {
      "targetEmail": "",
      "variables": [
        {
          "name": "",
          "pattern": "",
          "replace1": "-|-",
          "replace2": "-|-",
          "replace3": "-|-"
        }
      ]
    }
  ]
}

    Make sure to replace targetEmail with the email you are searching for.

    So a lot of people requested some documentation on how to make configs for Woxy, The config's are formed with JSON pretty simple and easy to edit for newbies too.

    Firstly, you will need to find the target email you are looking to check for amd change the following like in the format show to the right hand side...

    From this code"targetEmail": "",

    to this code for example "targetEmail": "email@facebook.com", by changing it to this, Woxy will search for emails from Facebook.com.

    So you are probably wondering what the other parts are that you need to fill in, don't worry we will explain that below.

    Name, Will be the name of the config that will display in Woxy to initiate the config when you type it so for example you would change it to the following: "name": "WoxyExample.com", so this code will make the config name WoxyExample.com when you are typing the name of the config in the program.

    ====================================================

    Author, this is pretty self explanitory this is where you will type your own username / name to clarify you made the config so for example: "author": "Dreamzje [ KEEPIT.ONLINE ]",.

    ====================================================

    Description, you just place a simple discription of what the config does here, it will also display next to the name of the config in Woxy, so for example: "description": "Test Format As Proof",.

    ====================================================

    to this code for example "targetEmail": "email@facebook.com", by changing it to this, Woxy will search for emails from Facebook.com.

    Variables

    The following code is how a configuration should look when finished but remove the \ in the pattern and all of the replace sections of the code from the first span:

    {
  "General": {
    "name": "Venmo",
    "author": "Dreamzje [ KEEPIT.ONLINE ]",
    "description": "Capture Venmo Payments",
	"onceEnabled": "false"
  },
  "Capture": [
    {
      "targetEmail": "venmo@venmo.com",
      "variables": [
		{
          "name": "Payment",
          "pattern": "<\span style=\"color:#009200;float:right;\">([^a]*)<\/span>",
          "replace1": "<\span style=\"color:#009200;float:right;\">|-",
          "replace2": "<\/span>|-",
		  "replace3": "-|-"
        }
      ]
    }
  ]
}

    So now you're going to be wondering what the hell you do with the variables, don't worry because in this next step we will go through each section individually and explain which section of a variable does what.

    So let's behin with:

    ====================================================

    Name, this section is just the name of the section you are capturing, for example if I wanted to capture payments from the emails it would look like the following: "name": "Payment",.

    ====================================================

    Pattern, So here is where the regex101.com comes in handy as you will be using REGEX to capture the data from the .html format of an email, here we are going to parse the payment method from the email as like in the following: "pattern": "<\span style=\"color:#009200;float:right;\">([^a]*)<\/span>", I've had to add an extra \ in the initial span code for it to act as code within the site, you will not need to do this while creating an actual config.

    ====================================================

    Replace1, in this section we are just using replace to replace the html format that surrounds the REGEX code so it would leave us with just the regex parsed code like following: "replace1": "<\span style=\"color:#009200;float:right;\">|-", yet again like in the pattern I've had to add the extra \ in the span to make the code work on site without breaking into the html formats just ignore that. So what we have done here in the replace is copy the first part before the REGEX so the html format surrounding the REGEX and replaced it with nothing. As you noticed in the format example code in the format section of the docs it had -|- which replaced nothing with nothing here we replaced the first html before the regex with nothing.

    ====================================================

    Replace2, this section we are just going to do the same as Replace1: but this time backwards, we are going to replace the html code AFTER the REGEX this time so for instance as shown in the preview to the right "replace2": "<\/span>|-", We are replacing the end span with nothing, now this should leave us with just a pure capture now we have removed the html surrounding the REGEX so your outputted response should look something like this when it's capturing data now Payment: £200.

    ====================================================

    Replace3, The Replace3 section is just incase you randomly get some html still showing in your capture for instance you have a html formatted BR or P or even a SPAN before or after your capture you can just use the third replace section to remove that extended html from the capture format so for instance If we have a random extra span before the capture we would change the Replace3 to the following, "replace3": "<\/span>|-" yet again removing the first \ from the span as it's not needed in the config just for me here as otherwise it would fuck with my html formats.

    Remember — Always test your REGEX first on regex101.com

    Short Terms and Descriptions

    Here is a small table that describes all the sections and variables with name | desctiption for those who are too lazy to read the whole documentation.
    Short Terms Short Description
    Name Configuration Name ( site name ).
    Author Author of The Config's Name.
    Description Small description of what the config does, what it captures.
    Once Enabled Will search for a certain email only once.
    targetEmail The email you are targeting to fetch emails from ( email@amazon.com, email@facebook.com ).
    Name Name of the variable. something you are trying to capture ( price, subscription, game, item, purchase ).
    Pattern The REGEX pattern match would be placed in this section ( regex101.com for testing).
    Replace1 Here is where you would replace the html surrounding the REGEX code with nothing so for instance HTML HERE |-The | means a split and the - means replace with nothing.
    Replace2 Here is where you would replace the html surrounding the REGEX code with nothing so for instance HTML HERE |-The | means a split and the - means replace with nothing.
    Replace3 Here is where you would replace the html surrounding the REGEX code with nothing so for instance HTML HERE |-The | means a split and the - means replace with nothing.
